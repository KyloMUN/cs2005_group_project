    5. Manual Grading
      1. Instructor can modify grades.

        Inital grading can be reviewed on a answer-by-answer basis and altered by a user with instructor priviledges.

        author:brown; date: Mon Jan 7 11:21:30 NST 2019

        Lecture Note #14. Use case Review Submission use case.

        Log in as instructor, modify student grades.

        This is done in the Manual Grading module under the function 'change_question_grade_for _student', essentially this method is accessed through the quiz system by choosing which question grade you would like to change, and you can return to the edit menu and edit as many grades as you would like, and each time having them stored in the system.

      2. Instructor can modify answers.

        Inital grading can be reviewed on a answer-by-answer basis and a student's answer can be updated and re-graded by a user with instructor priviledges.

        author:brown; date: Mon Jan 7 11:21:30 NST 2019

        Lecture Note #14. Use case Review Submission use case.

        Log in as instructor, modify student answer, see it regraded.

        This requirement is also covered in the function 'change_question_grade_for_student'. Since this requirement is very much similar to the previous requirement, we found it would be easier to have the requirement be dealt with by the same function that changes the grade, but with an added option to present a newAnswer you would like to overwrite the current answer with. If a newAnswer is provided, the fuction will overwrite the current answer with your provided newAnswer.

      3. Instructor can change quiz answers.

        The quiz may have had a bad question. Correct answer to a quiz item and re-grade the entire class.

        author:brown; date: Mon Jan 7 11:21:30 NST 2019

        Lecture Note #14.

        Log in as instructor, modify quiz answer, see if grades updated.

        This is covered in the function 'change_answer'. However, instead of editing currently existing quiz answers, the function instead appends the correct version of the answer to the list of currently correct answers. This is done to prevent conflicts between students who may have already completed the question and received a correct mark, but perhaps removing one of the answers would dock them a mark unfairly.

      4. Instructor can change quiz results.

        Instructor may want to adjust class overall or curve the quiz results, by giving a bonus or adjustment to the entire class of students.

        author:brown; date: Mon Jan 7 11:21:30 NST 2019

        Lecture Note #14.

        Log in as instructor, modify everyone's grade on a quiz; check individual student results.

        This is done in the 'change_grade' function in the module. The way the grading modules work together prevents a sure fire way of changing the grade of an entire class all at once, so changing an entire class' grades would be done through our react-based quiz system, perhaps by running a list of results through the function multiple times. But, where this was not deemed necessary, the function takes in a single student's result object and the instructor can provide them with a newGrade to boost, curve, or decrease their mark accordingly.

      5. Manual modifications are logged with optional note.

        Manual changes need to be reviewable, so it is possible to tell where grades came from.

        author:brown; date: Mon Jan 7 11:21:30 NST 2019

        Lecture Note #14.

        Log in as instructor, make modifications, and examine log of previous changes.

        Rather than being confined to a single method, the option to leave a note logging your changes is present within every function in the module. Where some may find this a case of code duplication, it was found that keeping the types of notes separate from eachother helps decouple the methods within the module, which helps the entire team.
